ABOUT THE TEST

The test will be approximately 50% conceptual/short-answer and 50% coding questions. In other words, 50% of the test will be devoted to true or false, fill in the black, definitions, short-response questions. The other 50% will be writing code similar to what you do on quizzes (look at quiz #3 for a good example). Expect one question that asks you to explain how to do a simple thing (draw a square, make cereal, tie a shoe, etc). These questions want LOTS of details. Also expect some questions to make you write a function based on pseudocode or to rewrite a given function. My advice is to begin with the conceptual portion first; If you don't know the answer to something here, skip it and move on to the rest of the conceptual and coding questions. Chances are something will jog your memory later on.

REVIEW QUESTIONS

As with any coding questions, there are multiple ways to solve a problem. The key is to find the most efficient algorithm. For example, if you needed to write a function that determines if a number is even, you could write LOTS of if statements comparing the number to every even number ever. But obviously, that is not effecient at all. Sample answers are provided to all the questions.

Question 1

Write a function called sum_of_odds that returns the sum of only the odd integers passed into the function. The function should accept three integer values as parameters. So if I pass in 12, 3 and 5, the result would be 8. If the integers 12, 4, and 20 are passed in, the result would be zero. ( 5 points)
